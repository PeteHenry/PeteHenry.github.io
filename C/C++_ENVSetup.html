使用SublimeText和MinGW构建轻量级C/C++编译器的开发环境设置 最近因为学C++的缘故，需要在电脑上安装C++的运行环境。而自己的笔记本已经有turbo C++和Microsoft Visual C++ 6.0这些编译C程序的开发工具了，不过虽然这些编译器虽然用起来很方便，但是在使用上感觉turbo C++界面很难看，一些熟悉的功能它没有，跟不上时代；VC++又太大了，界面也不习惯。而自己熟悉的就有Sublime Text，知道Sublime Text支持C的编译，但是还是需要电脑支持生成可执行文件的编译环境，这里就使用了MinGW。

这里我们最终要实现的目的是：当在Sublime Text编写的C代码后，通过Ctrl+B能构建成功，Ctrl+shift+B能正常运行，但要成功，则需要有G++（C++编译器），而我们安装MinGW，就是让它提供G++...

四、.sublime-build 这里注意在SublimeText 3之前的版本，可能需要增加.sublime-build文件

比如在Sublime Text 2中Tools –> Build System –> New Build System…

然后将以下内容粘贴过去

16

{ "cmd": ["g++", "${file}", "-o", "${filepath}/${filebasename}"], "fileregex": "^(..[^:]):([0-9]+):?([0-9]+)?:? (.)$", "workingdir": "${filepath}", "selector": "source.c, source.c++", "shell": true, "variants": [ { "name": "Run", "cmd": ["cmd", "/c", "g++", "${file}", "-o", "${filepath}/${filebasename}", "&&", "cmd", "/c", "${filepath}/${filebasename}"] }, { "name": "RunInCommand", "cmd": ["cmd", "/c", "g++", "${file}", "-o", "${filepath}/${filebasename}", "&&", "start", "cmd", "/c", "${filepath}/${filebasename} & pause"] } ] } 然后保存，保存的名字自己随便起，比如：C.sublime-build，注意后缀一定为.sublime-build。

该文件被保存在Sublime Text 2目录下的Data\Packages\User中。

然后在Tools –> Build System中选中该文件。

重启Sublime text 2 即可开始编译C了。
